# Speaker Notes — SAFEWATCH Mode

> 発表原稿・デモ説明原稿・MTG説明のドラフト。話し言葉ベース。適宜自分の言葉に直す。

## 発表原稿ドラフト

### 冒頭
「本日はコース3の卒業課題に向けたチーム開発のデモをお見せします。
まず最初にお断りしておくと、これは完成品ではなく、チームで認識を合わせるための"叩き台デモ"です。
テーマは『SAFEWATCH Mode ― 安全監視つきカスタムフライトモード』です。」

### デモ説明
「ArduCopter に SAFEWATCH という独自フライトモードを追加します。制御自体は既存の Stabilize をベースにしていて、
新しく作るのは"監視と警告"の部分です。今からSITL、つまり実機なしのシミュレータで動かします。
モードを SAFEWATCH に切り替えると、地上局にメッセージが出ます。
そのまま機体の姿勢や高度を動かして、あらかじめ決めた閾値を超えると、WARNING が地上局に表示されます。
最後にログでその挙動を確認します。」

### 技術説明
「技術的なポイントは3つです。1つ目は、フライトコードに新しいモードをどう登録するか。
2つ目は、roll・pitch・高度・アーム状態といった値をフライトコード側からどう取得するか。
3つ目は、警告メッセージを出しすぎないためのレート制限です。
コース3で学んだフライトコードの構造理解を、この監視ロジックの実装に活かしています。」

### 今後の開発方針
「今回のデモは警告表示までです。ここから必須機能・追加機能・やらないことをチームで線引きして、
閾値のパラメータ化や、監視項目の追加を検討していきます。異常時の自動アクションは今回スコープ外にしています。」

### 締め
「このデモをきっかけに、皆さんから追加アイデアや懸念点をいただきたいです。
特に『監視すべき項目』と『どこまでを卒業課題に含めるか』について意見をください。」
