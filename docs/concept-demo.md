# Concept Demo — 非エンジニア向け「住所で巡回」ドローン

> このドキュメントは「完成品の仕様書」ではなく、**チーム開発を始める前の認識合わせ用デモ**の説明です。
> 目的は「何を作るのか」をチームで見て、イメージをそろえること。
> ※ 方針の経緯は [design-decisions.md](design-decisions.md) を参照。

## デモの目的

- アイデアを、動く形（SITLデモ）で見せる
- チームメンバー全員が「作るもの」の完成イメージを共有する
- デモを叩き台にして、必須機能・不足・難易度・役割分担・ボトルネックを洗い出す
- 次回MTGで追加アイデア・懸念点を集める

## 背景

- コース3（デベロッパーコース・フライトコード編）の卒業課題としてチーム開発デモを提出する
- Mission Planner のような複雑なUIは、非エンジニアには壁が高い
- 「農地や施設の外周を毎回同じルートで見回る」は、**一度登録すれば繰り返し使える**巡回ニーズ
- コース3の評価対象である **ArduPilot フライトコードの理解** は、③安全機構で示す

## 解決したい課題

- 非エンジニアは、専門ツールなしにドローンの飛行を計画・実行できない
- 一方、自分で計画して飛ばすのは「素人ゆえの見落とし」で危険が伴う
- → **簡単に計画でき、かつ機体側が安全を見守る**仕組みが要る

## 製品の概要（3階建て）

| 層 | 役割 | この製品での実現 |
|---|---|---|
| ①計画 | どこを・どう飛ぶか | **住所を入れる → 周辺を自動で巡回ルート化** |
| ②実行 | 計画を機体に送って飛ばす | ミッション生成 → アップロード → **AUTOモードで飛行** |
| ③守る | 飛行中の安全監視 | 姿勢・高度・アーム状態を監視（**フライトコード側**） |

> **一言でいうと**: 住所を入れるだけで、非エンジニアがその周辺を安全に巡回飛行できる。

## デモで見せること（今回のスコープ）

1. 住所（または地名）を入力する
2. 住所から中心座標を取得し、周辺を**既定サイズで自動巡回**するミッションを生成
3. ミッションを SITL の ArduCopter にアップロード
4. **AUTOモード**で自動巡回飛行する様子を SITL で見せる
5. 飛行中、機体側の安全監視が働いていることを示す（姿勢・高度など）
6. ログ（tlog / メッセージ）で一連の動作を確認する

## 今回は見せないこと（意図的に除外 / ロードマップ）

- 実機での飛行（SITLのみ）
- 巡回ルートの細かいカスタム（サイズ・高度・周回数の手動調整）→ 将来
- 地図をなぞって正確な外周を描く（Lv2 地図UI）→ 将来
- 住所→AIによる畑輪郭の自動提案 → 将来
- 自然言語（「農地Aを見回って」）での指示 → 将来
- 異常時の自動アクション（自動RTL/着陸）→ 将来
- 本家 ArduPilot への Pull Request（自分のリポジトリ内で完結）

## 位置づけの明確化

| 観点 | 今回のデモ | 卒業課題の最終形（想定） |
|---|---|---|
| 計画 | 住所→既定サイズで自動巡回 | ルートのカスタム・地図なぞり・AI指示 |
| 安全 | 飛行中の監視 | 監視＋異常時の自動アクション |
| 実装 | 最小構成 or 一部モック | ①〜③を一貫実装 |
| 発表 | チーム内・認識合わせ | 講師・評価者向け |
