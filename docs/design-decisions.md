# Design Decisions Log

> チームでの検討・メンタリングを通じて決めた「何を・なぜ」の記録。
> 迷ったらここに戻る。決定は変わってよいが、**変えた理由を必ず残す**。

## この製品は何か（進化後のビジョン）

当初は「SAFEWATCH＝警告を出すカスタム飛行モード」だったが、検討の結果、
**製品の主役は"非エンジニアでも簡単にドローンの巡回飛行を計画・実行できること"**だと再定義した。

> **一言でいうと**: 住所を入れるだけで、非エンジニアがその周辺を安全に巡回飛行できるツール。

警告機能（旧SAFEWATCH）は「主役」ではなく、
**素人が作った飛行計画を機体側が見守る"安全機構"**として位置づけ直した。

## 全体像：3階建てモデル

| 層 | 役割 | 誰向け | 主な技術 |
|---|---|---|---|
| ① 計画（Plan） | 「どこを・どう飛ぶか」を簡単に指定 | 非エンジニアが触る | 住所→座標変換(ジオコーディング)、Python |
| ② 実行（Execute） | 計画をミッション化して機体に送り飛ばす | 自動（裏方） | pymavlink / dronekit / MAVLink / AUTOモード |
| ③ 守る（Protect） | 飛行中の異常を監視して安全に飛ばす | 自動（裏方） | ArduPilot フライトコード（＝コース3の評価対象の芯） |

- **魅せ場・情熱の対象** = ①（Mission Plannerを使わない簡単な計画体験）
- **課題としての芯・評価される所** = ③（フライトコード側の安全機構）
- ①だけでも③だけでもなく、両輪でひとつの物語にする。

## デモのスコープ（今回作るもの）

| 層 | デモに入れる | 後回し（ロードマップ） |
|---|---|---|
| ①計画 | 住所 → 周辺を**既定サイズで自動巡回**ルート生成 | ルート数値カスタム / 地図なぞり / AI自然言語 |
| ②実行 | ミッション生成→アップロード→AUTOで**SITL飛行** | 実機飛行 |
| ③守る | 飛行中の**安全監視**（姿勢・高度・アーム状態など） | 異常時の自動アクション |

**デモの一言**: 「住所を入れるだけで、非エンジニアがその周辺を安全に巡回飛行できる」をSITLで実演。

## 未解決・要確認（次の対話で詰める）

- **③安全機構の実装形が未確定**（重要）:
  巡回は標準の **AUTOモード** で飛ばす想定。しかし旧SAFEWATCHは「Stabilizeベースの独自モード」だった。
  AUTOで飛んでいる最中は別モードにできないため、「独自モードで監視」という当初案はそのままでは噛み合わない。
  → 安全機構を **(a) フライトコードの常時監視/フェイルセーフとして持つ** か、
    **(b) 独自モード自体が巡回も監視も行う** か、**(c) 併走監視スクリプト** かは、次回一緒に設計する。
- 住所ジオコーディングの提供元（無料のNominatim想定）と利用条件
- SITLのホーム座標を、ジオコーディング結果に合わせて動かす方法
- 既定の巡回サイズ・高度・周回数の初期値
- 提出形式（自分のrepo URL共有 / 指定repoへのPR）＝講師確認
- 本家ArduPilotへのPRは不要という理解でよいか＝講師確認
- ~~【評価基準】~~ **解決(D13)**: 明確な卒業要件は無く、各自が作りたいものを掘り下げる自由形式。
  → スコープの歯止めが外部に無いぶん、**完成可能性の自己管理**が最重要になった。
- **LuaかC++か**: 評価要件が無くなったので"証拠"ではなく「**フライトコードをどれだけ深く理解したいか**」で選ぶ。
  深く学びたい人がいるならC++、まず動かしたいならLua。担当者の興味で決めてよい。
- **③拡張の具体対象**: どの既存モード/仕組みを拡張するか（AUTO飛行中の監視 / pre-armへの自作検査 / RTL発動条件 など）。次回、実物コードを見て決める。

## 先行事例調査（Prior Art / 2026-07 時点）

構想した部品は、既存ツールでほぼ全て実現済みと確認した。
→ **新機能ではなく「簡単さ＋統合」で差別化する**根拠。車輪の再発明を避け、巨人の肩に乗る。

| 構想部品 | 既存？ | どこに |
|---|---|---|
| 多角形を描く→巡回ミッション | ✅ ある | QGC「Survey」、Mission Planner「Polygon→Auto WP」「WP Circle(外周巡回)」 |
| ジオフェンス（範囲外で自動帰還） | ✅ ある | ArduPilot標準 |
| 計画を検査して離陸を拒否 | ✅ 枠組みあり | Pre-armチェック群＋公式 `arming-checks.lua`（警告/arming拒否を選べる） |
| 飛行中の姿勢監視 | ✅ ある | 姿勢系フェイルセーフ＋Luaテレメトリ監視 |
| 外周巡回そのもの | ✅ ある | fence-follow / dwell点 / WP Circle |

**空いている席**＝「非エンジニアが迷わないわかりやすさ」。既存ツールは高機能だが初心者には複雑。ここが我々の主戦場。
