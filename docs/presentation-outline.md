## スライドごとの流用元

- 製品・決定の経緯 → `design-decisions.md`（D番号を引用すると説得力が出る）
- 安全設計・しきい値根拠 → `safety-design.md`（§8 範囲上限の根拠 など）
- デモ手順 → `demo-script.md` ／ 検証の証跡 → `sitl-test-log.md`
- 体制・進め方 → `role-assignment-draft.md` / `team-process.md`
- 話す文言 → `speaker-notes.md`

## 作成時の注意

- 文字を詰めすぎない（1スライド1メッセージ）
- **デモが主役**。スライドはデモの前後を支える役割
- 「実データで動いている」ことを繰り返し強調（当日の天気⚠すら証拠になる）
- 弱点も正直に: 建物高さ・地形はモック（明示済み）、実機は未検証 — 隠さず「次の一手」として語る
