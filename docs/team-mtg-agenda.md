## 次回MTGの目的

- **動いている一気通貫デモ**（①計画→②実行→③守る）を全員で見て、現状の到達点をそろえる
- ここからの**深掘り分担**（特に③のC++担当）を決める
- 卒業発表までのマイルストーンを引く
- 提出形式（URL共有 / PR）の確認状況を共有する

## タイムボックス（目安 60分）

| 時間 | 内容 |
|---|---|
| 0-5分 | 目的共有・今日のゴール確認 |
| 5-15分 | デモ（demo-script.md）: 住所→安全チェック(実データ)→SITL飛行→日誌 |
| 15-35分 | 議論①②（下記論点） |
| 35-50分 | 決定 → design-decisions.md にD番号で追記 |
| 50-60分 | 役割確定・宿題割当（role-assignment-draft.md「最初の一歩」）・次回日程 |

## デモを見る観点（メンバーへ事前共有）

- ①〜③が薄く一本つながって動いている（walking skeleton）。ここからは深掘りフェーズ
- 安全チェックは**実データ**（OSM・気象）。しきい値は根拠つき（safety-design.md §8）
- ③はStep1(設定自動投入)+Step2(Lua)まで済み。**Step3(C++)が残っている**
- 「非エンジニアが迷わないか」の目線で粗を探してほしい

## 議論したい論点

1. **③Step3（C++ pre-arm自作）を誰がやるか**・スコープは「1つ追加」で足りるか（D12）
2. 残モック（建物高さ・地形AGL）に手を出すか、モックのまま発表で語るか
3. 発表の型: 10分版アウトライン（presentation-outline.md）で行けるか
4. 実機に載せるか、SITLのみで完結させるか（安全・時間との相談）
5. スケジュール（発表日から逆算したマイルストーン）
6. 提出形式の確認状況（担当・期限）

## メンバーから集めたいアイデア

- 非エンジニア目線でUIが迷う箇所（実際に触ってもらう）
- 安全チェックの追加項目・しきい値への意見
- 発表で刺さりそうな見せ方（例: 京都御所⚠、中止→自動帰還、日誌DL）
- 想定される質問・弱点（評価者目線）
